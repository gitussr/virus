Chapter 1

What is a Computer Virus?

A computer virus is a type of malicious program that, like a biological virus, attaches itself to a host file and replicates itself to spread to other files and systems — often without the user knowing.

Biological Virus

  • Attaches to living cells
  • Replicates inside the host
  • Spreads through contact
  • Can cause harm or death
  • Needs a host to survive

Computer Virus

  • Attaches to program files
  • Replicates by copying code
  • Spreads via networks/drives
  • Can damage or steal data
  • Needs a program to activate

Key characteristics

Self-replication

A computer virus copies its own code into other programs or files automatically. This is the defining feature that sets it apart from other software.

Requires a host

Unlike worms, viruses cannot run on their own. They need to attach themselves to an existing executable file, document, or boot sector to activate.

Payload delivery

Once activated, most viruses carry a "payload" — malicious actions such as deleting files, displaying unwanted messages, or stealing data.

Trigger conditions

Many viruses lie dormant until a trigger condition is met — such as a specific date, a number of replications, or a certain user action.

Virus vs other threats

People often use "virus" to mean all malicious software, but technically a virus must replicate by attaching to a host. A worm spreads by itself without needing a host. Ransomware encrypts files for money. A Trojan disguises itself as safe software. Together, all of these fall under the umbrella term malware (malicious software).

Chapter 2

Types of Computer Viruses

Viruses come in many forms depending on where they hide, how they spread, and what they target. Here are the major categories you need to know.

Category 01

File Infector Virus

Attaches to executable files (.exe, .com). When the infected file runs, the virus activates and infects other executable files on the system.

Category 02

Boot Sector Virus

Infects the master boot record (MBR) of a storage drive. Loads before the OS starts, making it very hard to detect and remove.

Category 03

Macro Virus

Hides inside macro scripts in documents (Word, Excel). Spreads when infected documents are shared. Very common in the late 1990s.

Category 04

Polymorphic Virus

Changes its own code each time it replicates, making it very difficult for antivirus programs to detect using simple signature matching.

Category 05

Metamorphic Virus

The most advanced type — completely rewrites its own code on each infection. Even the behavior changes, making detection nearly impossible.

Category 06

Multipartite Virus

Attacks both the boot sector and executable files at the same time, making it spread faster and harder to fully clean from a system.

Category 07

Resident Virus

Loads itself into RAM and stays there. It can infect any file the OS accesses, even antivirus programs themselves.

Category 08

Non-resident Virus

Does not load into RAM. It infects files only when the infected file is executed, then exits. Simpler but still destructive.

Category 09

Overwrite Virus

Overwrites the original file content with its own code, permanently destroying the host file. The simplest and crudest type.

Category 10

Network Virus

Specifically designed to spread across computer networks via shared folders, email attachments, or network protocols.

Category 11

Stealth Virus

Actively hides its presence by intercepting OS calls that check for infected files and reporting fake "clean" data to the OS.

Category 12

Web Scripting Virus

Exploits vulnerabilities in web browsers or websites using scripts (JavaScript, HTML). Infects by visiting a compromised page.

Danger level indicators (1–5 dots)

Chapter 3

The Malware Family

A virus is just one member of the malware family. Here's a complete look at all the malicious software types that exist in the digital world.

Virus

Computer Virus

Attaches to a host file, replicates when the file is run, and can corrupt data, slow systems, or deliver damaging payloads.

Worm

Worm

Spreads across networks independently — no host file needed. Can replicate thousands of copies very quickly, consuming bandwidth and crashing systems.

Trojan

Trojan Horse

Disguises itself as legitimate software. Once installed, it opens backdoors for attackers, steals data, or installs other malware.

Ransomware

Ransomware

Encrypts the victim's files and demands payment (ransom) for the decryption key. One of the most financially damaging threats today.

Spyware

Spyware

Secretly monitors and collects user activity — keystrokes, browsing, passwords — and sends the data to a remote attacker.

Adware

Adware

Displays unwanted advertisements, often while tracking browsing habits. Usually not destructive but very annoying and invasive.

Rootkit

Rootkit

Hides deep within the OS, often at the kernel level. Gives attackers persistent, undetected control over the compromised system.

Keylogger

Keylogger

Records every keystroke a user types, capturing passwords, credit card numbers, messages, and other sensitive information.

Botnet

Botnet

Turns infected computers into "bots" controlled by an attacker. Networks of thousands of bots are used to launch DDoS attacks or send spam.

Fileless

Fileless Malware

Operates entirely in RAM without writing to the hard drive. Since no files are created, traditional antivirus tools struggle to detect it.

Scareware

Scareware

Tricks users into believing their device is infected, then sells fake security software. Classic social engineering.

Logic Bomb

Logic Bomb

Malicious code that stays dormant until a specific trigger (date, event, action) activates it — then executes its payload.

Chapter 4

A History of Computer Viruses

From a simple self-replicating experiment in 1971 to billion-dollar cyberattacks today — the evolution of computer viruses is a fascinating and cautionary tale.

1971
Creeper — The First
The Creeper program spread through ARPANET, displaying "I'm the creeper, catch me if you can!" It was actually an experimental self-replicating program, not malicious.
1982
Elk Cloner
The first virus to spread "in the wild" (outside a lab). It infected Apple II floppy disks and displayed a poem on every 50th boot.
1986
Brain — First PC Virus
Created by two Pakistani brothers, Brain infected the boot sector of IBM PC floppy disks. It was reportedly written to track pirated medical software.
1988
Morris Worm
One of the first worms distributed via the internet. It infected ~6,000 machines (about 10% of the internet then) and caused millions in damages.
1999
Melissa Virus
A macro virus that spread via infected Word documents in emails. It overwhelmed mail servers, causing $80 million in damages worldwide.
2000
ILOVEYOU
The "Love Bug" spread via email disguised as a love letter. It infected 45 million Windows machines in 10 days and caused $10 billion in damage.
2003
Blaster Worm
Exploited a Windows vulnerability to spread rapidly. When it activated, it triggered a continuous shutdown loop on infected computers.
2004
MyDoom
The fastest-spreading email worm ever recorded. At its peak, it was responsible for 1 in 4 emails sent worldwide.
2010
Stuxnet
A sophisticated cyberweapon that targeted Iran's nuclear centrifuges. Believed to be the first malware designed to cause real-world physical damage.
2017
WannaCry Ransomware
Infected 230,000 computers in 150 countries, encrypting files and demanding Bitcoin. Crippled the UK's National Health Service.
2020–present
Modern Threats
AI-powered malware, supply chain attacks (SolarWinds), ransomware-as-a-service, and state-sponsored cyberwarfare define the current era.
Chapter 5

How Computer Viruses Work

Every virus follows a lifecycle — from the moment it enters a system to when it executes its payload. Understanding this helps us defend against it.

The infection lifecycle

1
Entry
Virus enters via email, download, USB, network, or malicious website
2
Dormancy
Virus lies dormant, waiting for a trigger condition before activating
3
Propagation
Virus replicates itself, infecting more files, folders, or systems
4
Trigger
A condition (date, action, count) activates the virus from dormancy
5
Execution
The payload runs — deletes files, encrypts data, or contacts attacker

Common spread methods

Email attachments

The most common method. An infected .exe, Word doc, PDF, or ZIP is sent as an email attachment. The virus activates when the file is opened.

Malicious downloads

Free software, cracks, keygens, and torrents often bundle viruses. The user installs what they think is a game or tool — but it's a trojan.

USB drives

Viruses can infect USB drives and jump to any computer they're plugged into. The Stuxnet virus famously used USB drives to reach air-gapped nuclear systems.

Drive-by downloads

Just visiting a malicious or compromised website can trigger an automatic download and installation of malware without any user interaction.

Network shares

In corporate or school networks, viruses can spread through shared drives. Infecting one machine can quickly spread to hundreds of others.

Chapter 6

Effects of Computer Viruses

The impact of a virus can range from mildly annoying to catastrophically destructive — on individuals, organizations, and even entire nations.

Performance

System slowdowns

Viruses consume CPU, RAM, and disk resources while running in the background, making the system noticeably slower.

Data

File corruption or deletion

Many viruses overwrite, encrypt, or permanently delete files — documents, photos, videos, and even system files.

Privacy

Data theft

Viruses can steal passwords, banking credentials, personal photos, identity information, and send them to attackers.

Financial

Financial loss

Ransomware demands payment. Banking trojans steal money. Business downtime causes massive revenue losses. Globally, trillions of dollars are lost yearly.

Network

Network disruption

Worms consume all available network bandwidth, effectively taking down an organization's internet connection through sheer replication volume.

Reputation

Reputational damage

Businesses that suffer data breaches lose customer trust. Schools, hospitals, and governments face enormous public criticism after cyberattacks.

Chapter 7

Prevention & Protection

The best defence against viruses is prevention. Most infections are completely avoidable with good digital habits and the right tools.

Essential protection tips

  • 1
    Install reputable antivirus software — Use trusted antivirus programs like Windows Defender, Malwarebytes, or Avast. Keep them updated so they can detect the newest threats.
  • 2
    Keep your OS and software updated — Most viruses exploit known vulnerabilities. Software updates patch these holes. Enable automatic updates wherever possible.
  • 3
    Never open suspicious email attachments — If you don't recognize the sender, or the email seems odd even from someone you know, don't open attachments or click links.
  • 4
    Download software only from official sources — Avoid pirated software, cracks, or unofficial download sites. Use official app stores or the developer's own website.
  • 5
    Be careful with USB drives — Unknown USB drives can contain viruses. Don't plug in drives found in public places. Scan all external drives before opening them.
  • 6
    Back up your data regularly — If infected, backups let you restore your files. Use the 3-2-1 rule: 3 copies, 2 different media, 1 offsite (cloud).
  • 7
    Use strong, unique passwords — A strong password reduces the risk of a compromised account being used to spread malware to others.
  • 8
    Enable your firewall — A firewall monitors incoming and outgoing network traffic, blocking many types of network-based viruses and attacks.
  • 9
    Use HTTPS websites only — Look for the padlock in the browser address bar. HTTPS encrypts your connection, reducing the risk of drive-by malware injection.
  • 10
    Educate yourself and others — Most cyberattacks begin with human error. Learning to spot phishing emails, fake websites, and social engineering is your strongest defence.

How antivirus software works

Signature detection
Matches known virus "fingerprints" from a database of millions of signatures
Heuristic analysis
Detects suspicious behaviour patterns that resemble how viruses act
Sandboxing
Runs unknown files in an isolated virtual environment to observe their behaviour safely
Real-time scanning
Continuously monitors all file access, downloads, and program execution in the background
Cloud intelligence
Sends suspicious file hashes to cloud servers for community-driven, up-to-the-minute threat data

Chapter 9

Glossary of Key Terms

A quick-reference dictionary of all the important terms used in computer virus and cybersecurity topics.

Antivirus
Software designed to detect, prevent, and remove malicious programs from a computer system.
Botnet
A network of computers infected with malware and controlled remotely by an attacker for coordinated malicious activities.
DDoS Attack
Distributed Denial of Service — overwhelming a server with traffic from many infected machines until it crashes.
Encryption
The process of encoding data so only authorized parties can read it. Used by ransomware to lock victim files.
Exploit
A piece of code or technique that takes advantage of a vulnerability (bug) in software to cause unintended behavior.
Firewall
A security system that monitors and controls incoming and outgoing network traffic based on predefined rules.
Heuristic analysis
A method antivirus software uses to detect unknown viruses by identifying suspicious behaviors rather than known signatures.
Keylogger
Malware that records every keystroke made on a keyboard, capturing passwords, messages, and sensitive information.
Malware
An umbrella term for all malicious software, including viruses, worms, trojans, ransomware, spyware, and more.
Payload
The malicious action a virus or malware performs after activation — deleting files, stealing data, displaying messages, etc.
Phishing
A social engineering attack where fake emails, texts, or websites trick users into revealing credentials or downloading malware.
Quarantine
When antivirus software isolates an infected file, preventing it from running or spreading without deleting it immediately.
Ransomware
Malware that encrypts a victim's files and demands a ransom payment (usually in cryptocurrency) for the decryption key.
Rootkit
A type of malware that hides itself deep within the operating system, often at kernel level, to avoid detection.
Sandbox
An isolated virtual environment where suspicious programs can run safely to observe their behavior without risk to the real system.
Signature
A unique pattern of code that identifies a specific known virus, used by antivirus databases to detect that virus.
Social engineering
Psychological manipulation that tricks people into making security mistakes, like clicking malicious links or revealing passwords.
Spyware
Malware that secretly collects data about a user's activities and sends it to a remote attacker without the user's knowledge.
Trojan horse
Malware disguised as legitimate or useful software that performs hidden malicious actions once installed.
Vulnerability
A weakness or flaw in software, hardware, or a process that can be exploited by malware or attackers.
Worm
A type of malware that self-replicates and spreads across networks independently, without needing to attach to a host file.
Zero-day
A vulnerability that is unknown to the software vendor and has no patch available, making it extremely dangerous to exploit.
